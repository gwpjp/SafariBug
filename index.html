Safari is sometimes unable to change the style of an external style sheet because it changes the selectorText of all selectors to lower case. If I try to change a CSS tag that is written in Camel Case by clicking on the corresponding button, the code will not work. Therefore, the block does not change color. (Note the output to the console. It shows that the selector class name has been converted to all lower case.) However, if I try to change a CSS tag that is written in all lower case by clicking on its corresponding button, the code works and the block changes color. Both buttons work on other browsers such as Chrome and Firefox.
